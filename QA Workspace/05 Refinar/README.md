# 05 Refinar

Fila de **material bruto aguardando refinamento** — exports `.md` do Notion, texto colado de outra ferramenta, qualquer conjunto de coisas que precise virar card no padrão do vault ([[../Sistema/Contexto/FLUXOS#6. Refinar demanda já cadastrada (Notion → vault)|fluxo 6]]).

## Como usar

1. **Jogar o arquivo aqui**, do jeito que veio — bagunçado mesmo, sem arrumar nada. Se souber o número, prefixar no nome (`<SGV> - <qualquer coisa>.md`); se não, tanto faz.
2. **Pedir o refinamento** numa sessão com IA ("refina o que está no 05 Refinar"). O processo é o fluxo 6 + [[../Sistema/Skills/SKILL_BUGS#Modos de entrada|SKILL_BUGS — Modos de entrada]] (modo Contexto rico; com análise junto, a regra [[../Sistema/Skills/SKILL_BUGS#Bug com análise (causa raiz)|Bug com análise]]).
3. **Regra de ouro do refinamento**: nada é inventado. Cada informação do card nasce **extraída** do material, **inferida** como comportamento mais lógico (sempre listada explicitamente pra validação) ou **perguntada** quando ambígua — na dúvida, pergunta, não assume.
4. **Depois do card criado** (em `02 Demandas/<ambiente>/`) e da daily registrada (`📝 ... refinado(a)`), o arquivo bruto é **apagado** — mesma lógica das capturas legadas do Inbox: a fonte de verdade é a task no Notion, e o extrato estruturado vive no card.

Arquivo parado aqui é pendência: todo material nesta pasta deve ter um item correspondente em **A fazer hoje** da daily (`SGV-XXXX - Refinar (material em 05 Refinar/)`), pra fila não virar cemitério.
